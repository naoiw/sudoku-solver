# ページ仕様（SPEC）

## 目的

本リポジトリで実現したいこと：

- 数独の盤面を入力する
- **Solve** ボタンを押すと盤面を確定し、solver を実行して空いたマスを埋める
- 解をブラウザ上で表示する Web サービス

## プラットフォーム

- **開発・テスト**: モダンブラウザ
- **配信**: ブラウザベース
- **デプロイ**: GitHub Actions でリポジトリに push されたときに自動ビルド・デプロイ

## 画面構成

- **階層**: トップ階層の 1 ページのみ（ルーティングなし）

## 盤面

- **レイアウト**: 9×9 のマスを配置する
- **マスの操作**
  - マスをクリックすると、そのマスが選択された状態になる
  - マスが選択された状態で数字キー（1〜9）を入力すると、その数字がマスに入る
  - マスが選択された状態で **0** / **Space** / **Delete** / **Backspace** を押すと、そのマスの数字を削除する（空きマスにする）
  - 空きマスは**空白**で表示する（0 や記号は使わない）
- **マスへの数字入力があった際のチェック**
  - 入力があるたびに、すべてのマスでエラーチェックを行う（行・列・同一 3×3 ブロック内の重複を評価。空きマスはチェック対象外）
  - エラー判定のあった各マスは赤で表示し、`aria-invalid` で示す
- **解の表示**
  - solver で埋めた数字は、ユーザーが入力した数字と**色・スタイルで視覚的に区別**する

## Solve ボタン

- **無効化**: 不正なマス（行・列・同一 3×3 内で重複があるマス）が1つでもある間は、Solve ボタンは押せない（disabled）
- **動作**: Solve ボタンを押すと、盤面が確定し、solver が実行され、空いたマスを埋める
- **エラー表示**（いずれも**インライン**で表示）
  - **解がない場合**: 解が存在しない旨をエラー表示する
  - **入力データに誤りがある場合**（同じ行・列・ブロックに同じ数字があるなど）: 入力時チェックで不正マスを赤表示し、Solve は無効。Solve 実装後は実行時エラー表示も検討する

## その他 UI

- **リセット**: Reset ボタンで盤面を空にする。アンドゥ機能はなし
- **クリア**: Clear ボタンでソルバーが埋めた数字だけを削除する（ユーザー入力は残す）。ソルバーで埋めたマスが1つもないときは無効（disabled）
- **初期表示**: 空の盤面
- **UI 言語**: 日本語のみ

## アクセシビリティ

- 基本的なキーボード操作（マス選択・数字入力など）を想定する
- 不正なマスは色（赤）と `aria-invalid` の両方で表現する
